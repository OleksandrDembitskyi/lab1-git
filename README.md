# Lab1-Git — Лабораторні роботи з управління ІТ-проєктами

[![CI/CD Pipeline](https://github.com/OleksandrDembitskyi/lab1-git/actions/workflows/ci-cd.yml/badge.svg?branch=main)](https://github.com/OleksandrDembitskyi/lab1-git/actions/workflows/ci-cd.yml)

🌐 **Живий сайт:** [lab1-git-tau.vercel.app](https://lab1-git-tau.vercel.app)

## Проєкт
Базовий React-застосунок, створений для виконання серії лабораторних робіт. Слугує основою для вивчення Git, тестування, збірки, CI/CD та моніторингу.

## Технології
- React 19
- Vite 7
- JavaScript (ES6+)
- ESLint / Prettier
- **Тестування:** Vitest, Playwright

---

## 📁 Лабораторна робота №1 — Git та гілки
**Тема:** Управління кодом та організація робочого простору в Git

### Виконано:
- Створено структуру проєкту (`.gitignore`, `README.md`)
- Робота з гілками (`main`, `develop`, `feature/*`)
- Створено та змерджено Pull Request з код-рев'ю
- Вирішення merge-конфлікту

### Команди:
```bash
git branch                 # перегляд гілок
git checkout -b feature/initial-layout
git add .
git commit -m "feat: ..."
git push origin feature/initial-layout
```

---

## 🧪 Лабораторна робота №2 — Тестування
**Тема:** Забезпечення якості програмних обєктів через багаторівневе тестування

### Додано:
- Простий калькулятор (функції `add`, `subtract`, `multiply`, `divide`)
- **16 Unit-тестів** з Vitest (100% покриття бізнес-логіки)
- **5 E2E-тестів** з Playwright (перевірка UI)
- Аналіз покриття коду (Code Coverage)
- Навмисне внесення помилки (демонстрація падіння тесту)

### Команди для тестування:
```bash
npm run test:unit                 # запуск Unit-тестів
npm run test:unit:coverage        # покриття коду
npm run dev                       # запуск сервера
npx playwright test               # E2E-тести
npx playwright show-report        # перегляд звіту
```

## ⚙️ Лабораторна робота №3 — Автоматизація збірки
**Тема:** Автоматизація збірки проєкту та управління артефактами

### Виконано:
- Налаштовано скрипти автоматизації в `package.json` (`dev`, `build`, `preview`, `lint`)
- Додано змінні оточення (`.env` для розробки, `.env.production` для продакшену)
- Інтегровано змінні в інтерфейс: кольорова плашка з режимом, динамічний заголовок
- Виконано перевірку коду ESLint — 0 помилок, 0 попереджень
- Проведено аналіз артефактів збірки в папці `dist/` (хешування файлів, оптимізація)
- Оновлено `.gitignore` для захисту конфіденційних файлів

### Команди для роботи:
```bash
npm run build                 # створення продуктової збірки
npm run preview               # локальний перегляд збірки
npm run lint                  # перевірка коду лінтером
```

## 🚀 Лабораторна робота №4 — CI/CD
**Тема:** Безперервна інтеграція та автоматизація розгортання (CI/CD)

### Виконано:
- Налаштовано CI/CD пайплайн у GitHub Actions (`.github/workflows/ci-cd.yml`)
- Автоматична перевірка коду при push та Pull Request: лінтер, тести, збірка
- Перевірка реакції CI на помилки: навмисне зламано тест (червоний ❌) та виправлено (зелений ✅)
- Налаштовано автоматичний деплой на Vercel при push у гілку `main`
- Додано Branch Protection Rule для захисту гілки `main`
- Додано секрети `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`
- Створено та змерджено Pull Request #5 з `develop` в `main`
- Додано статус-бейдж CI/CD у шапку README

### Команди для перевірки:
```bash
npm run lint
npm run test:unit
npm run build
```

## 📊 Лабораторна робота №5 — Аналітика
**Тема:** Продуктова аналітика та управління на основі зібраних даних

### Виконано:
- Інтегровано продуктову аналітику **PostHog** у застосунок
- Налаштовано автоматичний збір подій (`$autocapture`, `$pageview`)
- Додано кастомні події для відстеження ключових дій:
  - `operation_selected` — вибір операції калькулятора
  - `calculation_performed` — виконання обчислення
  - `error_occurred` — помилка (ділення на нуль)
  - `clear_button_clicked` — натискання кнопки очищення
- Побудовано **воронку конверсії** (Calculator: conversion funnel) з трьох кроків
- Проаналізовано **записи сесій (Session Recordings)** та виявлено Rage clicks
- Створено **Feature Flag** (`show-clear-button`) для керування новою функціональністю
- Додано кнопку "Очистити", яка вмикається/вимикається через прапорець без зміни коду
- Налаштовано змінні оточення для продакшен-версії (`VITE_PUBLIC_POSTHOG_KEY`, `VITE_PUBLIC_POSTHOG_HOST`)
- Проведено перевірку роботи аналітики на живому сайті (`lab1-git-tau.vercel.app`)

### Команди для роботи:
```bash
npm install posthog-js @posthog/react        # встановлення PostHog
npm run dev                                   # запуск в режимі розробки
npm run build && npm run preview              # локальна перевірка продакшен-збірки
```

### Результати аналітики:
- **Загальна конверсія:** 66.67%
- **Найбільший відтік:** 33.33% між переглядом сторінки та вибором операції
- **Середній час до конверсії:** 1 хвилина 8 секунд
- **Feature flag:** кнопка "Очистити" з'являється/зникає відповідно до стану прапорця
